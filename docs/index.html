Send a voice instruction from your phone. Pam saves and transcribes it, runs Codex inside the correct project, preserves the result, and replies in Discord.

Discord is the control interface. The main outcome is work performed in project repositories, with a trackable record connecting each instruction to its result.

One installation, multiple projects

Each Discord channel maps to a separate local workspace and follows that project's own instructions and review process.

Setup

1. Install

git clone https://github.com/jlaffy/pam-discord.git
cd pam-discord
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
codex login

2. Add a Discord bot

Create a bot in the Discord Developer Portal, enable Message Content Intent, invite it to your server, and copy its private token.

3. Configure

cp .env.example .env
cp config.example.toml config.toml

Add the bot token to .env. Add approved user IDs, channel IDs, and their local project paths to config.toml.

4. Run

pam-discord --config config.toml

Send a short voice instruction in a mapped channel. Confirm that Pam posts the transcript and Codex result.

Cost: transcription runs locally. Codex uses your existing ChatGPT login rather than a separately billed model API key. Your Codex plan limits still apply.

Keep credentials and private archives out of Git. Allowlist users, use private Discord channels, and obtain consent before recording other people.
