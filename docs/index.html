From clone to first note.

You need Python 3.11+, Discord, a machine that stays online when you want Pam available, and an installed, authenticated Codex CLI.

Install Pam

Clone this repository, create an isolated environment, and install the package.

git clone <repository-url> pam-discord
cd pam-discord
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

Create the Discord bot

In the Discord Developer Portal, create an application and bot, enable Message Content Intent, then invite it to your server with permission to view channels, read history, send messages, and download attachments.

Add secrets and channel mappings

Copy the examples, place the bot token only in .env, and map channel IDs to absolute workspace paths in config.toml. Keep .env out of version control.

cp .env.example .env
cp config.example.toml config.toml
# Edit both files, then authenticate Codex if needed:
codex login

Start listening

Launch Pam, send a short test note in an approved channel, and confirm that a transcript and archive entry appear.

pam-discord
Cost and access. Local transcription does not require a hosted speech API. Codex CLI can use your ChatGPT authentication, so this setup does not require a separate model API key or API bill. Your ChatGPT plan and Codex usage limits still apply; this is not unlimited or guaranteed free. Discord’s native voice-message recording is primarily a mobile workflow—on desktop, attach an audio file or send text if your client does not offer recording.
